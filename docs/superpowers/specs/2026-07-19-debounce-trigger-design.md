# MUI × react-hook-form × use-debounce — デバウンス発火トリガー 設計書

日付: 2026-07-19
ステータス: 承認済み

## 目的

FAQ 提案 AI を呼び出すタイミング制御の PoC。3つのフォームフィールドがすべて埋まったら発火し、以後更新のたびにデバウンスされて発火するイベントハンドラを実装する。リアルタイム性と負荷軽減の両立が狙い。

API コール自体は実装しない。発火したことは画面上の履歴表示で確認する。

## 前提・制約

- 既存の `src/PocForm.tsx`(useForm + Controller ×3、フィールドは `trigger` / `question` / `category`)を土台にする
- デバウンスライブラリは [use-debounce](https://github.com/xnimorz/use-debounce) のみ使用する
- **ユーザーの文字入力はデバウンスしてはならない**(RHF の状態更新・画面反映は即時のまま)
- **submit はデバウンスされた値ではなく、その時点の入力値で行われなければならない**
- react-hook-form 7.82 の `subscribe` API(再レンダーなしの全フィールド購読)が利用可能

## 要件(確定事項)

- デバウンス待機時間: **2000ms**(最後の入力からの静止時間)。定数 `DEBOUNCE_WAIT_MS` として定義し調整可能にする
- 初回発火(3つ目が埋まった瞬間)も即時ではなく**デバウンス経由**
- どれかのフィールドが空(空白のみ含む)に戻ったら、**保留中のデバウンスをキャンセルし発火停止**。再び全フィールドが埋まったら発火を再開する
- 発火の可視化: 発火時刻とその時点の3フィールドの値をフォーム下に履歴として表示(新しい順)

## 設計(案A: subscribe + useDebouncedCallback)

### `src/useFaqSuggestionTrigger.ts`(新規)

「全フィールドが埋まったらデバウンス発火」のロジックを閉じ込めるカスタムフック。

- 引数: RHF の `subscribe`、発火時コールバック `(values: FormValues) => void`
- 内部:
  - `useDebouncedCallback(コールバック, DEBOUNCE_WAIT_MS)`
  - `useEffect` で `subscribe({ formState: { values: true }, callback })` を購読し、クリーンアップで unsubscribe
  - 購読コールバック内の判定: 3フィールドすべて `trim()` して非空なら `debounced(values)`、1つでも空なら `debounced.cancel()`
- 戻り値: `cancel`(submit 時にキャンセルするため)

### `src/PocForm.tsx`(変更)

- `useForm` から `subscribe` を取り出し、フックに渡す
- 発火時コールバック: 発火時刻と値を履歴 state に追記し、フォーム下に一覧表示(新しい順)。将来 AI API コールに置き換わる箇所であることをコメントで明示
- `onSubmit`: 従来どおり `handleSubmit` 経由(= 生の入力値で submit)。加えて submit 時に `cancel()` を呼び、保留中のデバウンス発火が submit 後に出るのを防ぐ
- Controller / TextField には一切手を入れない(入力非デバウンスの制約を構造的に保証)

### 挙動一覧

| 状況 | 挙動 |
|---|---|
| 3つ目のフィールドが埋まり、入力静止2秒 | 初回発火(履歴に1件追加) |
| 以後どこかを更新し、静止2秒 | 再発火 |
| 入力継続中(静止2秒未満) | 発火しない(タイマー延長) |
| どれかが空(または空白のみ)になる | 保留中の発火をキャンセルし、再び全部埋まるまで停止 |
| 送信ボタン | 現在の入力値で即 submit、保留中デバウンスはキャンセル |

## 検証方法

`pnpm dev` で表示し、上表の5パターンを目視確認する。テストは書かない(前回 PoC の方針を踏襲)。

## 採用しなかった案

- **案B: 各 Controller の onChange でデバウンス関数を呼ぶ**: トリガーが明示的だが、3フィールドに同じ配線を書くボイラープレートが多く、フィールド追加時に漏れやすい。
- **案C: useWatch + useDebounce(値のデバウンス)**: キーストロークごとに親が再レンダーされ、「空欄になったら保留中もキャンセル」の制御が値ベースでは表現しにくい。
